# Архив DFT расчётов на GPU

Расчёты в данной директории **приостановлены**.

## Причины

### `HEA_L12_GPAW_GPU.ipynb` и `HEA_L12_QE_32atoms_GPU.ipynb`
- **Проблема:** Ноутбуки запускаются, но вычисления выполняются на **CPU**, а не на GPU.
- **Техническая деталь:** Для работы GPU требуется ручная компиляция пакетов с поддержкой CUDA. Установка через `pip install` по умолчанию загружает CPU-версии библиотек.
- **Решение:** Время, необходимое на компиляцию и отладку GPU-сборок, превышает выгоду от ускорения для текущего объёма задач. Для генерации датасета MLIP эффективнее запустить больше расчётов параллельно на CPU, чем ждать настройки одного GPU.

### `HEA_L12_PySCF_GPU.ipynb`
- **Статус:** Работает корректно.
- **Проблема:** Потребляет больше ресурсов (VRAM/RAM), чем предоставляет бесплатный тариф Google Colab на старте сессии.

---

> **Актуальные расчёты:** Ведутся в CPU-версиях ноутбуков в родительской директории `DFT/`.
